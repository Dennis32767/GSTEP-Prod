# Security Policy

## Supported Versions

The GemStep protocol is under active development.
Only the latest commit on the `main` branch should be considered supported unless otherwise stated.

Do **not** assume deployed test contracts are production-ready.

---

## Reporting a Vulnerability

If you discover a potential security vulnerability, please report it responsibly.

**Do NOT disclose publicly** (including GitHub issues, Discord, or social media).

Instead, contact:

Please include:
- A clear description of the issue
- Steps to reproduce (if applicable)
- Affected contract(s) or module(s)
- Any proof-of-concept or test case

---

## Disclosure Process

1. We will acknowledge receipt within a reasonable timeframe.
2. We will investigate and validate the report.
3. If confirmed, we will work on a fix and coordinate disclosure.
4. Public disclosure will occur **after a fix is available**, where appropriate.

---

## Bug Bounties

At this time, GemStep does not operate a formal bug bounty program.

This may change in the future.

---

## Scope

In scope:
- Smart contracts in this repository
- Upgrade and governance logic
- Economic invariants and minting logic

Out of scope:
- Frontend UI bugs
- Social engineering attacks
- Third-party infrastructure failures
